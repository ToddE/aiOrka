## aiOrka Directory Structure

To implement the aiOrka library, follow this standard Kotlin Multiplatform (KMP) layout. This structure ensures that your configuration files are bundled correctly as shared resources.

## 📁 Library Layout (Shared Module)aiOrka-project

```
/
├── shared/                       # The core KMP module
│   ├── src/
│   │   ├── commonMain/           # Shared logic & resources
│   │   │   ├── kotlin/           # The "Orka" engine source code
│   │   │   │   └── org/aiorka/
│   │   │   │       ├── engine/   # SelectionEngine.kt
│   │   │   │       └── models/   # RegistryModels.kt
│   │   │   └── composeResources/ # Standard KMP Resource directory
│   │   │       └── files/        # Place default YAMLs here
│   │   │           ├── models-registry.yaml
│   │   │           └── aiOrka-default.yaml
│   │   ├── androidMain/          # Android-specific glue
│   │   ├── iosMain/              # iOS-specific glue
│   │   └── jsMain/               # Web/Node-specific glue
│   ├── build.gradle.kts          # KMP build configuration
│   └── ...
├── androidApp/                   # Sample Android app
├── iosApp/                       # Sample iOS app
└── build.gradle.kts              # Root build script
```

## 📍 Where to Place Your Files

### 1. Bundled (Default) Configuration
If you want the library to work "out of the box" with the comprehensive model list, place `models-registry.yaml` and a default `aiOrka.yaml` in: `shared/src/commonMain/composeResources/files/`

### 2. User-Provided Configuration
When developers use your library, they will likely want to point to their own files. In their project, they would typically place their custom `aiOrka.yaml` in their own platform-specific asset folders:
- Android: `src/main/assets/`
- iOS: Included in the main app bundle via Xcode.
- Backend/Desktop: In the root execution directory or `src/main/resources/`.

## 🏗️ KMP Resource Access Code
In your commonMain code, you will use the Res class (generated by Compose Multiplatform) or a simple ResourceReader to load these:

```
// Example loading the bundled registry
val registryContent = Res.readBytes("files/models-registry.yaml").decodeToString()
```

## 🐋 Asset Management (Branding)
Your icon should be placed in: `shared/src/commonMain/composeResources/drawable/aiOrka-icon.svg`
